### **CONSTITUTION.md**

**Project Constitution - Adaptive Life OS**

This document is law. Break it at your own risk.

**1. Core Principles**
- **Usefulness Over Polish** — It must be genuinely useful to *you* first. If you won’t use it daily, the project is trash.
- **Depth Over Breadth** — Better one module done exceptionally than five modules half-assed.
- **Production Mindset** — Code as if this will have 10,000 users and real money on the line.
- **User Friction is the Enemy** — Every extra click or second in daily logging must be justified.

**2. Quality Rules**
- All code must be TypeScript strict mode.
- No `any`. No disabled lint rules without comment.
- Every public UI component needs loading + error + empty states.
- All dates must respect user timezone.
- Performance: Main dashboard under 1.2s even with 2 years of data.
- UI must be addictive and motivating, never clinical or boring.

**3. Scope Discipline**
- If a feature is not listed in PROJECT.md Phase 1-3, it is out of scope until Phase 4+.
- No social features, no AI voice input, no Apple Health sync in v1.
- "Cool" is not a valid reason to add scope.

**4. Design & UX Constitution**
- Default to dark mode.
- Progress = green. Warnings = amber/red.
- Logging flow must be <20 seconds for 80% of daily use.
- Every chart must tell a clear story.
- Typography and spacing must be consistent and generous.

**5. Personal Usage Mandate**
- You *must* use this app daily during development.
- Every time you feel friction, fix it immediately and document it.
- Dogfood ruthlessly. This is non-negotiable.

**6. Portfolio Standards**
- Demo must be impressive in <3 minutes.
- Code must be clean enough for a senior engineer to respect.
- Include realistic seeded data.
- Excellent README with clear architecture explanation.

**7. When in Doubt**
- Ask: “Would this make a strong senior dev say ‘this guy knows what he’s doing’?”
- If answer is no → redo or remove.

**Violation of this constitution will result in a weaker portfolio piece and less personal life improvement.**
